core

注意

本Web版は公開デモ条件によるLAMMPS入力ファイル作成・操作確認用です。

社内材料名、顧客情報、未公開試験条件、開発テーマに紐づく条件は入力しないでください。

本Web版は解析実行、ファイルアップロード、外部保存を行いません。

社内条件を扱う場合は、所属組織の情報管理ルールに従い、会社管理下のローカル版または社内版を使用してください。

Web Safe Modeでは、現在CG系の公開デモ生成だけを有効にしています。AA/HybridはLocal版で有効化する機能として表示のみ残しています。
1. 入力Case / Molecule / Protocol を入力
2. 保存生成ファイルを同じ作業フォルダへ置く
3. 実行LAMMPS側で in.lammps を開く

操作例

まずこの順番で試す

LJ fluidを作る

  1. LJ fluidカードを選ぶ
  2. 初期値のまま生成
  3. まとめてZIPを保存
  4. ZIPを作業フォルダに展開
  5. lmp -in in.lammpsで実行
  6. dump.demo.lammpstrjをOVITOで開く

CG scaffoldを作る

  1. CG scaffoldカードを選ぶ
  2. bead数、chain数、pair係数を確認
  3. Protocolのrun stepsやtimestepを確認
  4. 生成してFilesへ進む
  5. 簡単に試すならin.lammpsを実行
  6. 配置/テンプレートを使うならpackmol.inp/system.ltを手動で使う

Protocol

どこに何を置いて、どう実行するか

  1. 作業フォルダを作る例: C:\lammps-demo\lj-fluid
  2. このWeb画面で生成するまずは初期値のまま 生成 を押す
  3. 生成ファイルを同じフォルダへ保存case.json, in.lammps, procedure.md、CG scaffoldでは packmol.inp, system.lt も必要に応じて保存
  4. LAMMPSで実行作業フォルダで lmp -in in.lammps
  5. 見たい場合生成された dump.demo.lammpstrj をOVITOで開く
手順表示
PowerShell例 
cd C:\lammps-demo\lj-fluid
lmp -in in.lammps

Run / Analysis

実行・解析はLocal版専用です

Web Safeでは無効

Web版はLocal版の機能制限版です。実行・解析系の操作は見える状態にしていますが、公開Webではクリックできません。

Local版で有効になるもの 外部ツールのパス設定、作業フォルダ作成、LAMMPS/PACKMOL/Moltemplate/Python/OVITO連携、結果フォルダ整理、log/dump/trajectory解析。

Settings

外部ツール設定はLocal版専用です

Web Safeでは無効

Web版ではローカルPCの実行ファイルや保存先を読み書きしません。設定欄はLocal版で有効になる項目として表示だけ残します。
